# Frame Data Spec — DrawCall & Render Pipeline

## Overview

In V4, the "compiler" is replaced by **systems** that produce and consume `DrawComponent`.

```
ECS Systems (time → resolve → source) → DrawComponent → render_sys → GPU
```

## DrawComponent (Runtime-only)

Attached to each visible entity by `source_sys`. Never serialized.

```rust
struct DrawComponent {
    draw_calls: Vec<DrawCall>,
    material_chain: Vec<Material>,
    texture_requests: Vec<TextureRequest>,
}
```

## DrawCall

Universal render primitive. All positions in **world space** (post-hierarchy cascade).

```rust
struct DrawCall {
    kind: DrawKind,
    // Transform (world space)
    x: f32, y: f32,
    width: f32, height: f32,
    rotation: f32,              // radians
    anchor_x: f32, anchor_y: f32,
    // Visual
    opacity: f32,
    blend_mode: BlendMode,
    color: [f32; 4],            // RGBA fill/tint
    // Content
    texture_key: Option<String>,
    shader: String,
    params: Vec<f32>,
    // Fit
    fit_mode: FitMode,
    intrinsic_width: f32,
    intrinsic_height: f32,
}

enum DrawKind {
    // Content (from source_sys)
    SolidRect,
    SolidEllipse,
    Texture,
    Text,
    // Editor overlays (from editor_sys)
    Outline,
    Gizmo,
    CameraFrame,
}
```

## TextureRequest

Generated by `source_sys` for assets that need on-demand loading.

```rust
enum TextureRequest {
    DecodeVideoFrame {
        key: String,
        asset_url: String,
        timestamp_secs: f64,
    },
    LoadImage {
        key: String,
        asset_url: String,
    },
}
```

## Render Pipeline (render_sys)

```
1. COLLECT:  Gather all DrawComponents from entities
2. SORT:     By entity layer, then by DrawCall order
3. PROJECT:  For each view, project world → screen via camera
             screen_pos = (world_pos - cam_pos) × scale + screen_center
4. MATERIAL: If entity has material_chain:
             a. Render base DrawCalls → intermediate RT
             b. Each material: input RT → shader → output RT
             c. Final RT → composite to main pass
5. COMPOSE:  Build FlatEntity list for GPU
6. SUBMIT:   RenderPass list → GPU → pixels
```

## Multi-View Rendering

```rust
struct ViewConfig {
    camera_id: String,
    screen_width: u32,
    screen_height: u32,
    // Editor overrides (pan/zoom)
    cam_override_x: Option<f32>,
    cam_override_y: Option<f32>,
    cam_override_w: Option<f32>,
    cam_override_h: Option<f32>,
    output_target: String,
    clear_color: [f32; 4],
}
```

Same DrawComponent data → multiple ViewConfigs → reuse texture cache.

```rust
// Editor: viewport + preview
tick(world, time, &[
    ViewConfig::editor("cam", pan, zoom, 800, 600),
    ViewConfig::preview("cam", 320, 180),
]);

// Export: full resolution, no editor overlays
tick(world, time, &[
    ViewConfig::export("cam", 1920, 1080),
]);
```

## RenderPass (GPU-level)

```rust
struct RenderPass {
    output: String,           // render target name
    pass_type: PassType,
    target_width: Option<u32>,
    target_height: Option<u32>,
}

enum PassType {
    Entities {
        entities: Vec<FlatEntity>,
        clear_color: [f32; 4],
    },
    Effect {
        shader: String,
        inputs: Vec<String>,
        params: Vec<f32>,
    },
}
```

## Data Flow Summary

```
Consumer → Scene JSON → load_scene()
         → tick(time, mode, views)
             Phase 1: time_sys
             Phase 2: animation_sys → rect_sys → hierarchy_sys
             Phase 3: source_sys → [editor_sys] → render_sys → GPU
         ← texture_requests → Consumer decodes → cache_image/video_frame → GPU textures
         ← pixels on canvas
```
